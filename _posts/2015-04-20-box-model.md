css盒模型是指具有 `margin,border,padding,content` 这些属性，且生效的元素以 **何种方式计算** 在页面中占据的大小。
盒模型有：W3C盒模型和IE盒模型两种


* W3C盒模型： `元素所占据的位置= margin + border + padding + content`
* IE盒模型：计算的属性和W3C一致，但是 `content` 包含了 `border + padding + content` 。

他们之间的区别就是关于``content``属性的计算
<img src="/images/1/model.png" alt="模型差异" />
测试例子：

~~~ html
<body>
	<style>
    	.box{
        	width: 100px;
            height: 200p;
            padding: 10px;
            margin: 20px;
            border: 5px solid red;
        }
    </style>
    <div id="box" class="box"></div>
    <script>
    	var $box = document.getElementById("box");
        //offsetWidth = content + scrollBar + padding + border; （scrollBar是滚动条）
        alert($box.offsetWidth); // ==> ie5:100 chrome:130
        //chrome : contentWidth = 130 - 5 * 2 - 10 * 2 = 100;
        //ie5	: contentWidth  = 100 - 0  = 100; //padding 和 border的值为“零”了，事实上他们只是被计算到了content内.
    </script>
</body>
~~~

###### 解决办法：加上doctype声明

~~~ html
// HTML5 (向后兼容，推荐使用):
<!DOCTYPE html>
~~~

但是在ie5中doctype是没有用的，还好ie5已死多年。
